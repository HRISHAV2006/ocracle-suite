# OCRacle — Frontend Architecture & Specification

**Stack:** React Native (Expo) + Next.js (Web)  
**Design System:** Custom — "EcoTruth" Design Language  
**Last Updated:** April 2026

---

## 1. Overview

The OCRacle frontend consists of two surfaces:

| Surface | Framework | Purpose |
|---|---|---|
| Mobile App | React Native (Expo SDK 51) | Primary user experience — camera scanning |
| Web Dashboard | Next.js 14 (App Router) | Vercel-hosted web version + browser extension |
| Chrome Extension | Manifest V3 | In-browser product scanning on e-commerce sites |

### 3.2 Key Screen Specifications

#### ScannerScreen
```
Layout: Full-screen camera view
Elements:
  - Camera viewport (full screen)
  - Scan reticle overlay (animated pulse)
  - Torch toggle (top-right)
  - "Type manually" link (bottom)
  - Brand logo (top-left, semi-transparent)

Camera Behavior:
  - Auto-focus on center
  - Barcode detected → haptic feedback → navigate to ScanResultScreen
  - OCR mode (manual text scan) → 2-second capture → loading state
```

#### ScanResultScreen
```
Layout: Scrollable card-based
Sections:
  1. Product Header Card
     - Product name + brand
     - Category + barcode
     - Scan timestamp

  2. Truth Score Card (HERO)
     - Large circular score dial (animated fill on load)
     - Score: "X.X / 10" in JetBrains Mono
     - Score label + color band
     - Animated reveal: score counts up from 0

  3. Sub-Scores Breakdown
     - Carbon Footprint: score + icon + label
     - Water Usage: score + icon + label
     - Chemical Toxicity: score + icon + label
     - Each has a horizontal progress bar

  4. Score Explanation
     - 2–3 bullet points explaining WHY this score
     - Generated by Claude API

  5. CTA Row
     - "See Alternatives" (primary)
     - "Ask Expert ₹49" (secondary)
     - "Save to Wishlist" (icon)

  6. Similar Products Carousel
     - 3 alternatives, horizontal scroll
```

#### ImpactDashboardScreen
```
Layout: Scrollable with sticky header
Sections:
  1. Total Impact Hero
     - Three large stat cards: CO₂, Plastic, Water
     - Animated number counters
     - Shareable card generator button

  2. Monthly Trend Chart
     - Line chart (Recharts / Victory Native)
     - Toggle: CO₂ / Plastic / Water

  3. Active Challenges
     - Challenge cards with progress bars
     - Countdown timer to challenge end

  4. Leaderboard Preview
     - Top 3 users this month
     - User's own rank
     - "View Full Leaderboard" link
```

## 7. API Integration Layer

### 7.1 API Client Setup

```typescript
// lib/api.ts
import axios from 'axios';

export const apiClient = axios.create({
  baseURL: process.env.EXPO_PUBLIC_API_URL || 'https://api.ocracle.com',
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json',
  },
});

// Auto-attach Firebase auth token
apiClient.interceptors.request.use(async (config) => {
  const token = await auth.currentUser?.getIdToken();
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});
```

### 7.2 Key API Calls

```typescript
// Scan product by barcode
const scanProduct = async (barcode: string): Promise<ScanResult> => {
  const { data } = await apiClient.post('/api/scan/barcode', { barcode });
  return data;
};

// Scan product by OCR text
const scanByText = async (ingredientText: string): Promise<ScanResult> => {
  const { data } = await apiClient.post('/api/scan/text', { text: ingredientText });
  return data;
};

// Get alternatives for a product
const getAlternatives = async (productId: string): Promise<Product[]> => {
  const { data } = await apiClient.get(`/api/products/${productId}/alternatives`);
  return data;
};

// Book expert consultation
const bookExpert = async (expertId: string, productId: string): Promise<Consultation> => {
  const { data } = await apiClient.post('/api/consultations', { expertId, productId });
  return data;
};
```

## 8. Performance Requirements

| Metric | Target |
|---|---|
| Time to first scan result | < 3 seconds |
| App startup (cold) | < 2 seconds |
| Camera viewfinder FPS | ≥ 30 FPS |
| Image bundle size | < 2 MB |
| Lighthouse score (web) | ≥ 90 |
| Offline support | Last 10 scans cached locally |

### 8.1 Optimization Strategies
- **Image lazy loading** on alternatives carousel
- **React Query caching** — same product scanned again = instant result
- **Skeleton screens** — all data states have loading skeletons
- **Code splitting** — heavy screens (camera, charts) loaded on demand
- **Expo EAS Build** — optimized native builds per platform

---

## 9. Accessibility (WCAG 2.1 AA)

- All interactive elements: minimum 44×44px touch target
- Color is never the only differentiator (score labels have icons + text)
- Screen reader support for Truth Score dial (aria-label with score and category)
- Dynamic font size support (iOS/Android accessibility settings)
- High contrast mode: score colors maintain 4.5:1 contrast ratio

## 11. Deployment

### Mobile
- **CI/CD:** GitHub Actions → EAS Build → App Store / Play Store
- **OTA Updates:** Expo Updates (hot fixes without app store review)
- **Environments:** development | staging | production

### Web (Next.js)
- **Hosting:** Vercel
- **Domain:** ocracle.com (production), staging.ocracle.com
- **Environment Variables:** Set in Vercel Dashboard
  ```
  NEXT_PUBLIC_API_URL=https://api.ocracle.com
  NEXT_PUBLIC_FIREBASE_CONFIG=...
  NEXT_PUBLIC_STRIPE_KEY=...
  ```

### Chrome Extension
- **Build:** webpack → dist/ folder
- **Distribution:** Chrome Web Store
- **Permissions:** activeTab, storage, identity
